The Problem
Before vs. After ShowRunner
AI assistants generate meeting deliverables, but the output needs significant manual work to become useful.

Before: Manual Process

  • ❌ Column names change every meeting ("Owner" vs "Assigned to")
  • ❌ AI silently truncates files (9 items discussed, 1 exported)
  • ❌ No carry-forward — open items vanish between days
  • ❌ 30-60 minutes per team per day on reconciliation
  • ❌ Sprint reports take hours to compile manually
  • ❌ No urgency tracking — overdue items go unnoticed
➔

After: ShowRunner

  • ✅ Flexible schema mapping resolves 12+ column name variants
  • ✅ Cross-references Word docs to recover all truncated items
  • ✅ Automatic carry-forward with status change detection
  • ✅ Single command processes all teams in ~2 minutes
  • ✅ Sprint reports auto-generated with metrics and health
  • ✅ Urgency flags computed daily (Overdue / Due Today / On Track)

Capabilities
Key Features
Built to handle real-world messiness in AI-generated project data.
🔍

Flexible Schema Mapping

12+ column name aliases per field. Whether Copilot says "Owner", "Assigned to", or "Owner (Full Name)" — it resolves correctly.

🔨

Truncation Recovery

When AI exports drop items silently, the cross-reference engine parses Word docs and recovers missing data using token-overlap similarity.

🔄

Carry-Forward Logic

Open items automatically persist across days. Status transitions are detected and flagged as New, Changed, or Unchanged.

⚠️

Urgency Classification

Every item gets a computed flag: Overdue, Due Today, Due This Week, or On Track. Sprint health rolls up to Green/Yellow/Red.

📨

Teams Integration

Posts daily digests via Incoming Webhooks (Adaptive Cards) or Microsoft Graph API. Dry-run mode for testing.

📈

Sprint Reports

Aggregates daily data into executive summaries with completion metrics, recurring blocker detection, and leadership recommendations.
